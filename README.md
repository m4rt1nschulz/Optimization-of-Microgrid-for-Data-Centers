# Optimization of Microgrid Composition for Sustainable Data Centers

## Overview
This repository contains a Python-based multi-objective optimization framework designed to size renewable microgrids (Solar, Wind, and Battery Storage) powering a 1 MW Data Center. 

As the energy demand of data centers grows, co-locating them with renewable microgrids is becoming a critical solution. However, engineers face a constant trade-off: minimizing grid dependency (Operational Emissions) requires building massive local infrastructure (Embodied Emissions). 

This project leverages co-simulation and genetic algorithms to automate the sizing of these systems, evaluating the complex trade-offs between OPEX CO2, CAPEX CO2, and Financial Cost.

## Key Features
* **Multi-Objective Optimization:** Utilizes the NSGA-II genetic algorithm (via `pymoo`) to generate 2D and 3D Pareto fronts, mapping the trade-offs between financial cost and environmental impact.
* **High-Fidelity Co-Simulation:** Built on top of the `vessim` testbed, simulating power flows at an hourly resolution over a full year (8760 hours).
* **Dynamic Battery Degradation:** Integrates the **Rainflow-counting algorithm** to dynamically calculate battery fatigue and lifespan based on Depth of Discharge (DoD) micro-cycles, heavily penalizing systems that wear out batteries prematurely.
* **Geopolitical Case Study (FR vs. UK):** Compares optimal grid designs for Gif-sur-Yvette (France) and London (UK) using real 2022 meteorological and carbon intensity data.

## Repository Structure

The project is organized into the following directories:

* `/codes`: Contains all Python scripts, including the multi-objective optimization algorithms (`multi_optimization_2obj.py`, etc.), the Vessim environment setup, and the custom battery logic.
* `/data`: Hourly datasets for 2022, including grid prices, carbon intensity (RTE for France, NESO for the UK), and SAM-extrapolated wind/solar profiles.
* `/docs`: Project documentation, including the final academic report and core reference literature.
* `/media`: Visual outputs generated by the scripts, categorized into:
  * `2ObjPareto/`: 2D plots illustrating the CAPEX CO2 vs OPEX CO2 trade-off.
  * `3ObjPareto/`: 3D Pareto projections including Total Annual Cost.
  * `rainflow/`: Battery cycle depth histograms.
  * `weather/`: Contextual seasonal energy generation profiles.
* `/drafts`: Initial baseline testing scripts and sandbox environments.
